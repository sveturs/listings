# Паспорт компонента: StorefrontHeader

## Метаданные
- **Путь**: `/frontend/svetu/src/components/storefronts/public/StorefrontHeader.tsx`
- **Роль**: Заголовок публичной страницы витрины магазина
- **Тип**: Презентационный компонент с интерактивностью
- **Размер**: 123 строки

## Назначение
Отображает визуальный заголовок витрины магазина, включая баннер, логотип, название, тип бизнеса и ключевую статистику. Компонент создает профессиональное первое впечатление о магазине.

## Props
```typescript
interface StorefrontHeaderProps {
  storefront: Storefront;        // Данные витрины
  isOwner: boolean;              // Является ли текущий пользователь владельцем
  onImageClick: (images: string[], index: number) => void; // Обработчик клика по изображениям
}
```

## Зависимости
### Внешние
- `next/image` - оптимизированные изображения
- `next-intl` - локализация
- `@/types/storefront` - типы данных

### Внутренние
- Нет прямых зависимостей от других компонентов

## Управление состоянием
### Локальное состояние
- Нет внутреннего состояния

### Внешние данные
- `storefront` - данные витрины передаются через props
- `isOwner` - флаг владельца (не используется в текущей версии)

## Бизнес-логика
1. **Отображение изображений**:
   - Баннер по умолчанию: `/storefront-banner-default.jpg`
   - Логотип по умолчанию: `/storefront-logo-default.png`
   - Клик по изображениям открывает галерею

2. **Типы бизнеса**:
   - retail - Розничная торговля
   - service - Услуги
   - restaurant - Рестораны
   - other - Другое

3. **Статистика**:
   - Средний рейтинг (с точностью до 0.1)
   - Количество товаров
   - Количество отзывов

4. **Верификация**:
   - Отображается badge для верифицированных магазинов

## UI структура
```
<div> // Контейнер
  ├── <div> // Баннер (320px высота)
  │   ├── <Image> // Изображение баннера с hover эффектом
  │   ├── <div> // Градиент overlay
  │   └── <div> // Badge верификации (если есть)
  │
  └── <div> // Информационная панель
      ├── <div> // Логотип (128x128px, -mt-16)
      │   └── <Image> // С рамкой и hover эффектом
      │
      ├── <div> // Название и категория
      │   ├── <h1> // Название магазина
      │   └── <p> // Тип бизнеса
      │
      └── <div> // Быстрая статистика
          ├── Рейтинг
          ├── Количество товаров
          └── Количество отзывов
```

## Примеры использования
```typescript
// На публичной странице витрины
<StorefrontHeader 
  storefront={storefrontData}
  isOwner={user?.id === storefrontData.user_id}
  onImageClick={handleImageGallery}
/>
```

## Известные особенности
### Позитивные
- Адаптивный дизайн (мобильный/десктоп)
- Оптимизированная загрузка изображений
- Плавные hover эффекты
- Поддержка дефолтных изображений

### Технический долг
- Параметр `isOwner` не используется (префикс `_`)
- Нет обработки ошибок загрузки изображений
- Фиксированная высота баннера может обрезать изображения

### Возможные улучшения
- Добавить функционал для владельца (кнопка редактирования)
- Реализовать lazy loading для изображений
- Добавить skeleton loader во время загрузки
- Сделать высоту баннера настраиваемой
