# Паспорт компонента: StorefrontInfo

## Метаданные
- **Путь**: `/frontend/svetu/src/components/storefronts/public/StorefrontInfo.tsx`
- **Роль**: Информационная панель витрины магазина
- **Тип**: Презентационный компонент
- **Размер**: 177 строк

## Назначение
Отображает детальную контактную информацию и режим работы витрины магазина. Компонент предоставляет все необходимые данные для связи с магазином и понимания его графика работы.

## Props
```typescript
interface StorefrontInfoProps {
  storefront: Storefront; // Полные данные витрины
}
```

## Зависимости
### Внешние
- `next-intl` - локализация
- `date-fns` - форматирование дат
- `date-fns/locale` - локали для дат (ru, enUS)
- `@/types/storefront` - типы данных

### Внутренние
- Нет прямых зависимостей

## Управление состоянием
### Локальное состояние
- Нет внутреннего состояния

### Внешние данные
- `storefront` - все данные получаются из props
- `locale` - текущая локаль из контекста

## Бизнес-логика
1. **Определение текущего дня**:
   - Автоматическое определение сегодняшнего дня недели
   - Поиск часов работы для текущего дня
   - Поддержка сокращенных названий дней (mon, tue, etc.)

2. **Форматирование часов работы**:
   - Отображение "Открыто" с временем работы
   - Отображение "Закрыто" для выходных дней
   - Обработка отсутствующих данных о времени

3. **Контактная информация**:
   - Телефон с кликабельной ссылкой `tel:`
   - Email с ссылкой `mailto:`
   - Внешний сайт с target="_blank"
   - Социальные сети (Facebook, Instagram)

4. **Время существования**:
   - Расчет времени с момента создания витрины
   - Локализованное отображение (ru/en)

## Известные особенности
### Позитивные
- Условное отображение только доступной информации
- Интерактивные элементы (телефон, email, ссылки)
- Поддержка локализации для дат
- Адаптивный дизайн

### Технический долг
- Сложная логика определения текущего дня недели
- Предполагается конкретная структура данных hours
- Нет валидации форматов телефона/email

### Возможные улучшения
- Добавить копирование контактов в буфер обмена
- Показывать статус "Скоро откроется/закроется"
- Добавить карту для отображения локации
- Поддержка большего количества социальных сетей
- Кеширование расчета текущего дня
