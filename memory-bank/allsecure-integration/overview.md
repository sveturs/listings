# AllSecure Payment Gateway - Полный обзор и анализ

## О компании AllSecure

### Основная информация
- **Основана**: 2001 год
- **Опыт работы**: 23+ года в сфере платежных систем
- **Статус**: Глобальный провайдер платежных услуг (Payment Service Provider)
- **Сертификация**: PCI DSS Level 1 (высший уровень безопасности)
- **Домены**: 
  - Сербия: https://www.allsecure.rs/
  - Глобальный: https://www.allsecure.eu/
- **GitHub**: https://github.com/allsecure-pay

### Позиционирование
AllSecure позиционируется как единственный реальный выбор для e-commerce бизнеса и единственный партнер, который полностью независим от банков и в первую очередь защищает интересы мерчанта.

## Продукты и услуги

### 1. Payment Gateway (Платежный шлюз)
- **PCI DSS Level 1** сертифицированный шлюз
- Обработка в множественных рынках и валютах через единую платформу
- Умное и экономически эффективное решение
- Acquirer agnostic (независимость от эквайеров)

### 2. Open Payment Platform
- Единый RESTful API для доступа ко всему функционалу платформы
- Frontend виджеты для быстрой интеграции
- Полная гибкость в настройке

### 3. SecurePay Widget
- SAQ-A совместимое встраиваемое JS Payment Widget решение
- Обеспечивает безопасность и простоту интеграции
- Минимальные требования к PCI compliance

### 4. Mobile SDKs
- iOS SDK для нативной интеграции
- Android SDK для нативной интеграции
- Полностью кастомизируемый checkout в мобильных приложениях
- Токенизация карточных данных в Card Vault

### 5. Server-to-Server REST API
- Гибкие back-office операции
- Capture, Void, Refund и другие операции
- Полностью гибкие workflow для frontend и backend обработки

## Техническая документация API

### API Versions
- **API V3 (JSON)**: https://asxgw.com/documentation/apiv3 - рекомендуемая версия
- **API V2 (XML)**: https://asxgw.com/documentation/api - устаревшая, будет отключена

### Base URL
```
https://asxgw.com/api/v3
```

### Аутентификация
```http
Authorization: Basic YW55QXBpVXNlcjpteVBhc3N3b3Jk
Content-Type: application/json
```
- Использует BASIC Authentication
- Credentials кодируются в Base64
- Опциональная подпись запроса через HMAC SHA2-512

### Методы транзакций
1. **Debit** - прямое списание средств
2. **Preauthorize** - резервирование средств
3. **Capture** - списание зарезервированных средств
4. **Void** - отмена транзакции
5. **Register** - регистрация карты для будущих платежей
6. **Deregister** - удаление зарегистрированной карты
7. **Refund** - возврат средств
8. **Payout** - выплата средств
9. **Incremental Authorization** - увеличение суммы авторизации

### Статусы транзакций
- **FINISHED** - транзакция успешно завершена
- **ERROR** - транзакция завершилась ошибкой
- **REDIRECT** - требуется редирект пользователя для завершения
- **PENDING** - транзакция в обработке

### Формат ответов
```json
{
  "success": true/false,
  "errorMessage": "описание ошибки",
  "errorCode": "код ошибки",
  "adapterMessage": "сообщение от банка",
  "adapterCode": "код от банка"
}
```

### Webhooks и уведомления
- Асинхронные уведомления о статусе
- Success notifications
- Error notifications  
- Chargeback notifications
- Account updater notifications

## Безопасность и комплаенс

### Сертификации
- **PCI DSS Level 1** - высший уровень сертификации
- **SAQ-A compliance** для widget интеграций
- **TLS 1.2+** обязательно для всех соединений

### Функции безопасности
- Токенизация карточных данных
- Card Vault для безопасного хранения
- Система предотвращения рисков (risk prevention)
- Опциональная подпись запросов

### Compliance требования
- PCI DSS соответствие обязательно
- Необходима регистрация как мерчант
- KYC (Know Your Customer) процедуры
- Соответствие местному законодательству Сербии
