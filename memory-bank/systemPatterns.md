# Архитектурные паттерны и ключевые решения

## Архитектура
- Микросервисный подход для отдельных доменов (баланс, маркетплейс, уведомления и т.д.)
- Использование REST API для взаимодействия между сервисами
- Хранение данных в PostgreSQL, файлов — в MinIO, поиск — в OpenSearch
- Использование очередей для асинхронных задач (например, отправка уведомлений)

## Ключевые технические решения
- Разделение backend и frontend
- Использование Docker и docker-compose для деплоя и локальной разработки
- Логирование и мониторинг через централизованные сервисы

## Паттерны
- Dependency Injection для сервисов
- Repository pattern для работы с БД

## Взаимосвязь компонентов
- Каждый домен изолирован, взаимодействие через API
- Общие сервисы (например, пользователи, платежи) доступны для всех доменов через отдельные сервисы
