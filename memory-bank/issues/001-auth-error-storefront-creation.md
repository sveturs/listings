# Проблема с авторизацией при создании витрины

## Дата обнаружения: 2025-06-24
## Статус: Не решена

## Описание проблемы
При попытке опубликовать витрину после прохождения всех 7 шагов wizard'а создания витрины появилось сообщение об ошибке: `users.auth.error.authentication_required`.

## Контекст
1. **Браузер**: Пользователь был авторизован в системе (имел доступ к странице /profile/storefronts)
2. **Процесс**: Успешно прошел все 7 шагов создания витрины:
   - Шаг 1: Основная информация (название, URL, описание)
   - Шаг 2: Данные о бизнесе (пропущен)
   - Шаг 3: Местоположение (адрес, город, индекс)
   - Шаг 4: Часы работы (использованы дефолтные)
   - Шаг 5: Оплата и доставка (выбраны наличные и карта)
   - Шаг 6: Настройка персонала (пропущен)
   - Шаг 7: Просмотр и публикация

3. **Ошибка**: При нажатии кнопки "Опубликовать" появилось сообщение `users.auth.error.authentication_required`

## Детали ошибки
- **Место**: Страница `/ru/create-storefront`, шаг 7 из 7
- **Действие**: Клик по кнопке "Опубликовать" (ref=e545)
- **Сообщение**: `users.auth.error.authentication_required`

## Что странно
- Без авторизации невозможно было бы дойти до страницы создания витрины
- Все предыдущие шаги прошли успешно
- Ошибка появилась только на финальном этапе публикации

## Что нужно проверить
1. Истек ли токен авторизации за время прохождения wizard'а
2. Корректно ли работает middleware авторизации для endpoint создания витрины
3. Передается ли токен в запросе при публикации витрины
4. Есть ли особые требования к правам для создания витрины
