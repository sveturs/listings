### Структура БД
- Таблица `districts` содержит поля:
  - `boundary` - geometry(Polygon,4326) для хранения полигонов границ
  - `center_point` - geometry(Point,4326) для центральной точки

## Потенциальные источники геоданных

### 1. OpenStreetMap (OSM)
- **Преимущества**: 
  - Бесплатный и открытый
  - Активное сообщество в Сербии
  - Доступны административные границы (admin_level=10 для районов)
  - Экспорт в различных форматах
- **Недостатки**:
  - Качество данных может варьироваться
  - Требуется проверка актуальности

### 2. Республички геодетски завод (RGZ)
- **Преимущества**:
  - Официальные данные от государственного органа
  - Высокая точность
  - Актуальные границы
- **Недостатки**:
  - Может быть платным
  - Возможны ограничения лицензии
  - Сложность доступа

### 3. GADM (Global Administrative Areas)
- **Преимущества**:
  - Бесплатный для некоммерческого использования
  - Единообразные данные для всех стран
  - Доступны различные уровни детализации
- **Недостатки**:
  - Может не содержать городских районов
  - Обновляется не так часто

### 4. Natural Earth
- **Преимущества**:
  - Открытые данные
  - Оптимизированы для картографии
- **Недостатки**:
  - Только крупные административные единицы
  - Не подходит для городских районов

## Рекомендуемый подход

1. **Первичный источник**: OpenStreetMap
   - Использовать Overpass API для загрузки границ
   - Фильтровать по admin_level и названию города
   
2. **Резервный источник**: GADM
   - Для районов, отсутствующих в OSM
   
3. **Инструменты обработки**:
   - Python + GeoPandas для обработки данных
   - QGIS для визуализации и ручной проверки
   - ogr2ogr для конвертации форматов
