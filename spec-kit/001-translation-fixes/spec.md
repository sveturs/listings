# Feature Specification: Translation System Fixes

## User Scenarios

- Как пользователь, я хочу видеть корректные переводы на всех страницах сайта
- Как пользователь из Сербии, я хочу видеть интерфейс на сербском языке без английских фрагментов
- Как русскоговорящий пользователь, я хочу видеть интерфейс полностью на русском языке
- Как администратор, я хочу быть уверен что все тексты в системе локализованы

## Requirements

### Functional Requirements

- FR-001: Система ДОЛЖНА отображать все интерфейсные элементы на выбранном языке без fallback на английский
- FR-002: Система ДОЛЖНА поддерживать три языка: английский (en), русский (ru), сербский (sr)
- FR-003: Система ДОЛЖНА корректно обрабатывать вложенные структуры переводов
- FR-004: Пользователь ДОЛЖЕН видеть переводы для всех модулей системы
- FR-005: Система ДОЛЖНА логировать отсутствующие переводы в режиме разработки

### Non-Functional Requirements

- NFR-001: Покрытие переводами ДОЛЖНО составлять минимум 95% для всех языков
- NFR-002: Время загрузки переводов НЕ ДОЛЖНО превышать 100ms
- NFR-003: Система ДОЛЖНА поддерживать hot-reload переводов в dev режиме

## Acceptance Criteria

### Для модуля marketplace
- Given пользователь на странице маркетплейса
- When выбран русский или сербский язык
- Then все 45+ ключей модуля должны отображаться на выбранном языке

### Для модуля common
- Given пользователь на любой странице сайта
- When используются общие элементы интерфейса
- Then все 35+ ключей модуля common должны быть переведены

### Для модуля admin
- Given администратор в админ панели
- When просматривает любой раздел админки
- Then все элементы управления должны быть на выбранном языке

### Общие критерии
- Given система запущена в production
- When пользователь выбирает любой из поддерживаемых языков
- Then не должно быть ошибок типа MISSING_MESSAGE в консоли

## Dependencies

- next-intl библиотека для интернационализации
- Существующая модульная структура переводов
- JSON файлы переводов в src/messages/

## Scope

### В рамках проекта:
- Добавление 242 недостающих переводов
- Исправление структурных проблем в JSON файлах
- Валидация консистентности между языками
- Настройка автоматической проверки переводов

### Вне рамок проекта:
- Изменение архитектуры системы переводов
- Добавление новых языков
- Рефакторинг компонентов

## Risks

- RISK-001: Неправильные переводы могут ухудшить UX для пользователей
  - Mitigation: Привлечение носителей языка для проверки
  
- RISK-002: Добавление переводов может сломать существующую функциональность
  - Mitigation: Поэтапное добавление с тестированием

## Metrics

- Процент покрытия переводами (target: >95%)
- Количество ошибок MISSING_MESSAGE в production (target: 0)
- Время выполнения всех исправлений (target: 2 недели)

## Notes

По результатам аудита найдено 242 недостающих перевода, распределенных по модулям:
- marketplace: 45+ ключей
- common: 35+ ключей
- admin: 30+ ключей
- Остальные модули: 130+ ключей суммарно

Система архитектурно правильно построена, основная задача - добавить недостающий контент.
