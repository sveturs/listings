## 2. Бизнес-модель и финансовые потоки

### 2.1 Источники дохода платформы

#### A. Комиссии с продаж (основной доход)
- **C2C сделки**: 2-5% от суммы сделки
- **B2C витрины**: 5-7% от суммы продажи

#### B. Платные услуги для пользователей
- **Продвижение объявлений**:
  - Поднятие в топ: 200-500 RSD
  - VIP статус: 1,000-3,000 RSD/месяц
  - Выделение цветом: 300-700 RSD
- **Подписки для продавцов (витрины)**:
  - Starter: 1,500 RSD/месяц
  - Professional: 5,000 RSD/месяц
  - Enterprise: 15,000 RSD/месяц
- **Дополнительные услуги**:
  - Premium поддержка
  - Аналитика и статистика

### 2.2 Типы финансовых операций

#### Сценарий 1: Оплата внутренних услуг маркетплейса
**Поток**: Покупатель → Платежный шлюз → Маркетплейс

**Примеры операций**:
- Оплата подписки на витрину
- Покупка услуг продвижения
- Оплата VIP статуса
- Пополнение внутреннего баланса для оплаты им в будущем платных услуг маркетплейса

**Требования**:
- Мгновенное зачисление средств
- Без холдирования
- Автоматическая фискализация
- Поддержка рекуррентных платежей для подписок (возможность автоматически списывать деньги с карты клиента на регулярной основе)

#### Сценарий 2: B2C продажи (витрины)
**Поток**: Покупатель → Эквайринг → Эскроу → Split-платежи

**Распределение средств**:
- Продавец: 85-90% (минус комиссия маркетплейса)
- Маркетплейс: 3-5%
- Логистика: 0.01-10% (если применимо)
- ПВЗ: 0.1-2% (если применимо)
- Платежная система: 0.01-2%

**Требования**:
- Холдирование средств на 7-14 дней (защита покупателя)
- Автоматический split после подтверждения доставки
- Возможность частичного возврата
- Интеграция с логистическими партнерами

#### Сценарий 3: C2C продажи (физические лица)
**Поток**: Покупатель → Эквайринг → Эскроу → Split-платежи

**Распределение средств**:
- Продавец: 85-95% (минус комиссия маркетплейса)
- Маркетплейс: 2-5%
- Логистика: 0.01-10% (если продавец выбрал доставку)
- ПВЗ: 0.1-2% (если продавец сдал товар в ПВЗ)
- Платежная система: 0.01-2%

**Особенности**:
- Продавец - физическое лицо без ПИБ
- Холдирование средств до подтверждения получения
- Выплата на банковский счет или карту продавца
- Возможность использования ПВЗ и доставки

**Требования**:
- KYC верификация для продавцов (суммы > 10,000 RSD)
- Поддержка выплат на карты физлиц


## 3. Технические требования

### 3.1 Методы оплаты (приём средств)
**Необходимые на старте**:
- Банковские карты (Visa, Mastercard, Dina)
- IPS QR (моментальные платежи через QR-код - стандарт НБС Сербии)
- Банковский перевод

### 3.2 Методы выплат (вывод средств)
**Необходимые на старте**:
- Банковский перевод на счет юрлица (для B2C продавцов)
- Перевод на карту физлица (для C2C продавцов)
- IPS переводы (моментальные переводы между банками Сербии)

### 3.3 Функциональные возможности

#### Обязательные функции:
1. **Эскроу (холдирование средств)**
   - Настраиваемый период холдирования (1-30 дней)
   - Автоматический релиз по выполненным условиям
   - Возможность продления холда при спорах

2. **Split-платежи (мультисплит)**
   - Распределение на 3+ получателей
   - Настраиваемые правила распределения
   - Автоматический расчет комиссий

3. **Возвраты и отмены**
   - Полный возврат
   - Частичный возврат
   - Отмена до захвата средств

4. **Рекуррентные платежи**
   - Для подписок
   - Автоматическое продление
   - Управление подписками

5. **Маркетплейс-функции**
   - Онбординг продавцов (KYC/KYB)
   - Управление балансами
   - Массовые выплаты
   - Реестры транзакций

#### Желательные функции:
**Токенизация карт** - сохранение карт покупателей в зашифрованном виде (токенах) для быстрых повторных платежей без повторного ввода данных карты

**3D Secure 2.0** - дополнительная проверка подлинности платежа через SMS-код или приложение банка, снижает риски мошенничества и перекладывает ответственность на банк-эмитент

**Антифрод система** - автоматическая проверка платежей на признаки мошенничества (необычная геолокация, множественные попытки, подозрительные паттерны)

**Динамическая конвертация валют** - возможность показывать цены и принимать платежи в валюте карты покупателя с автоматической конвертацией

**Мобильный SDK** - готовые библиотеки для интеграции платежей в мобильные приложения (iOS/Android)

**Webhook уведомления** - автоматические HTTP-запросы от платежной системы к нашему серверу при изменении статуса платежа

### 3.4 Интеграция

#### API требования:
- REST API (предпочтительно) или другой современный протокол
- Webhook для статусов платежей
- Sandbox (тестовая среда) для разработки
- Документация API (английский или сербский)
- Примеры интеграции

**Примечание**: Мы готовы адаптироваться под API платежной системы. Наш стек: Go (backend), JavaScript/TypeScript (frontend)

#### Безопасность:
**PCI DSS compliance** - сертификация соответствия стандартам безопасности платежных карт. Означает, что платежная система защищает данные карт по международным стандартам

**TLS 1.2+ для всех соединений** - использование современных протоколов шифрования для всех соединений между нашим сервером и платежной системой

**Токенизация чувствительных данных** - замена реальных данных карт на уникальные токены, которые бесполезны для злоумышленников

**Двухфакторная аутентификация для админов** - дополнительная защита административных панелей через SMS или приложение-аутентификатор

**IP whitelist** - ограничение доступа к API только с разрешенных IP-адресов наших серверов

## 4. Юридические и compliance требования

### 4.1 Лицензирование (требования к платежному оператору)

**Лицензия НБС на платежные услуги** - официальное разрешение от Народного банка Сербии на осуществление платежных операций. Подтверждает легальность и надежность оператора

**Право на осуществление эквайринга** - лицензия на прием платежей с банковских карт от имени торговцев (в нашем случае - маркетплейса)

**Право на эскроу-услуги** - разрешение на временное хранение средств покупателя до выполнения условий сделки (критично для защиты покупателей)

**Право на выплаты физическим лицам** - лицензия на осуществление выплат на карты и счета физических лиц, которые не являются предпринимателями (необходимо для C2C)

### 4.2 Соответствие регуляторным требованиям

**Соответствие закону Сербии о платежных услугах** - работа в рамках закона "Zakon o platnim uslugama", регулирующего платежные операции в Сербии

**AML/CFT процедуры** - Anti-Money Laundering (противодействие отмыванию денег) и Counter Financing of Terrorism (противодействие финансированию терроризма). Система мер для предотвращения использования платформы в незаконных целях

**KYC/KYB верификация** - Know Your Customer (знай своего клиента) для физлиц и Know Your Business для юрлиц. Проверка личности пользователей для предотвращения мошенничества

**Фискализация транзакций** - автоматическая передача данных о транзакциях в налоговую службу Сербии в соответствии с требованиями закона о фискализации

**GDPR compliance** - соответствие европейскому регламенту о защите персональных данных (General Data Protection Regulation), обязательному в Сербии
