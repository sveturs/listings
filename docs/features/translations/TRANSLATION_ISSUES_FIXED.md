# Исправление проблем с переводами объявлений

## Обнаруженные проблемы

1. **Ошибка SQL запроса**: В коде искалась колонка `en_translation`, которой нет в таблице `attribute_option_translations`
2. **Неверный API путь**: Frontend обращался к `/api/v1/profile` вместо `/api/v1/users/profile`
3. **Превышен лимит Google Translate API**: Бесплатный аккаунт достиг лимита запросов
4. **Отсутствие переводов опций атрибутов**: В БД не было переводов для значений select полей
5. **Неправильный SQL запрос для загрузки переводов**: Искал переводы в неправильной таблице

## Выполненные исправления

### 1. Backend исправления

- **Файл**: `backend/internal/proj/marketplace/storage/opensearch/repository.go`
  - Заменен `en_translation` на `ru_translation` в SQL запросе
  
- **Файл**: `backend/internal/proj/marketplace/storage/postgres/marketplace.go`
  - Переписан SQL запрос для получения переводов из таблицы `attribute_option_translations`

### 2. Frontend исправления

- **Файл**: `frontend/svetu/src/app/[locale]/create-listing-ai/page.tsx`
  - Исправлен путь API с `/api/v1/profile` на `/api/v1/users/profile`

### 3. Добавлены переводы

- **Миграция 000148**: Добавлены переводы для всех опций атрибутов (condition, color, transmission и т.д.)
- **Вручную добавлены переводы** для объявления ID 26 на сербский и английский языки

## Результат

- ✅ Исправлены ошибки SQL запросов
- ✅ Исправлен путь API на frontend
- ✅ Добавлены переводы опций атрибутов
- ✅ Backend успешно перезапущен
- ✅ Переводы теперь должны корректно отображаться
