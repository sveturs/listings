# Руководство по управлению атрибутами категорий

## 1. Общее описание функционала

Система управления атрибутами категорий позволяет гибко настраивать характеристики товаров и услуг для любой категории в маркетплейсе Sve Tu Platform. Этот инструмент даёт возможность:

- Создавать уникальные наборы атрибутов для каждой категории
- Настраивать различные типы полей ввода (текст, числа, списки, булевы значения и т.д.)
- Добавлять обязательные и необязательные атрибуты
- Настраивать пользовательский интерфейс для форм создания и редактирования объявлений
- Контролировать порядок отображения атрибутов в интерфейсе
- Поддерживать многоязычные значения для атрибутов и их параметров

Правильная настройка атрибутов категорий критически важна для создания качественного пользовательского опыта и обеспечения релевантности поиска на платформе.

## 2. Пошаговая инструкция по работе с интерфейсом

### Вход в панель управления атрибутами

1. Войдите в административный интерфейс платформы
2. Перейдите в раздел "Управление категориями и атрибутами"

### Выбор категории

1. В левой части экрана отображается дерево категорий
2. Раскройте нужный раздел, кликнув на иконку раскрытия (треугольник)
3. Выберите конкретную категорию, щелкнув на её название
4. Информация о выбранной категории и её атрибуты отобразятся в правой части экрана

### Просмотр текущих атрибутов категории

После выбора категории в правой панели отображается:
- Основная информация о категории (название, ID, путь)
- Таблица атрибутов, связанных с этой категорией
- Для каждого атрибута показываются:
  - Название атрибута
  - Тип данных
  - Обязательность заполнения
  - Параметры отображения
  - Действия (редактирование, удаление)

### Добавление атрибута к категории

1. Нажмите кнопку "Добавить атрибут" в разделе атрибутов выбранной категории
2. В появившейся форме заполните базовые параметры:
   - Название атрибута (для каждого поддерживаемого языка)
   - Тип атрибута (строка, число, список и т.д.)
   - Обязательность заполнения (да/нет)
3. Настройте параметры отображения:
   - Порядок отображения (число)
   - Видимость в поиске (да/нет)
   - Видимость в деталях объявления (да/нет)
4. Для типа "Список" добавьте варианты значений:
   - Нажмите "Добавить вариант"
   - Введите название для каждого языка
   - Повторите для всех необходимых вариантов
5. Нажмите "Сохранить" для привязки атрибута к категории

### Настройка расширенных параметров атрибута

1. Выберите существующий атрибут из таблицы и нажмите "Редактировать"
2. В расширенных настройках можно указать:
   - Единицы измерения (для числовых атрибутов)
   - Минимальное и максимальное значение
   - Шаг изменения значения
   - Валидационные правила (регулярные выражения для текстовых полей)
   - Подсказки для пользователя
3. Для атрибутов типа "Изображение" настройте:
   - Максимальное количество изображений
   - Требования к размеру и формату
4. Для атрибутов типа "Местоположение" настройте:
   - Тип данных (координаты, адрес, область на карте)
   - Параметры отображения карты
5. Нажмите "Сохранить" после внесения изменений

### Настройка пользовательского UI для категории

1. В разделе категории перейдите на вкладку "Пользовательский интерфейс"
2. Настройте группировку атрибутов:
   - Создайте секции для логического разделения атрибутов
   - Перетащите атрибуты в соответствующие секции
   - Настройте заголовки секций для каждого языка
3. Настройте условную логику отображения (если поддерживается):
   - Укажите правила, при каких значениях одних атрибутов показывать другие
4. Просмотрите предварительный вид формы
5. Сохраните настройки интерфейса

## 3. Рекомендации по настройке атрибутов для разных типов категорий

### Недвижимость

**Обязательные атрибуты:**
- Тип недвижимости (список: квартира, дом, комната и т.д.)
- Количество комнат (число)
- Площадь (число с единицей измерения)
- Этаж и этажность здания (числа)
- Адрес и местоположение на карте

**Дополнительные атрибуты:**
- Наличие мебели (булево значение)
- Тип отопления (список)
- Года постройки (число)
- Парковка (булево или список типов)
- Наличие балкона/террасы (булево или число)

### Транспорт

**Обязательные атрибуты:**
- Марка и модель (связанные списки)
- Год выпуска (число)
- Тип кузова (список)
- Пробег (число с единицей измерения)
- Тип двигателя (список)

**Дополнительные атрибуты:**
- Мощность двигателя (число с единицей измерения)
- Тип трансмиссии (список)
- Цвет (список)
- Состояние (список: новый, б/у, требует ремонта)
- Наличие сервисной книжки (булево)

### Электроника

**Обязательные атрибуты:**
- Тип устройства (список)
- Бренд (список или строка)
- Модель (строка)
- Состояние (список: новый, б/у)

**Дополнительные атрибуты:**
- Технические характеристики (зависят от типа устройства)
- Комплектация (список с множественным выбором)
- Гарантийный срок (число с единицей измерения)
- Цвет (список)

### Услуги

**Обязательные атрибуты:**
- Тип услуги (список)
- Описание (текстовое поле)
- Длительность (число с единицей измерения или список)
- Местоположение (адрес или область на карте)

**Дополнительные атрибуты:**
- Квалификация (строка или список)
- Опыт работы (число с единицей измерения)
- Возможность выезда (булево)
- Время работы (специальный тип - расписание)

## 4. Примеры типичных настроек

### Пример 1: Аренда квартиры

**Базовые атрибуты:**
- Тип жилья: Список [Квартира, Студия, Комната, Дом]
- Количество комнат: Число (мин: 1, шаг: 1)
- Общая площадь: Число с единицей измерения "м²"
- Этаж: Число
- Всего этажей в доме: Число
- Адрес: Текст + карта

**Особенности:**
- Аренда длительность: Список [Посуточно, Долгосрочно]
- Кто сдает: Список [Собственник, Агентство]
- Залог: Число с валютой
- Коммунальные платежи: Список [Включены, Не включены, Частично включены]

**Удобства (множественный выбор):**
- Интернет
- Кондиционер
- Стиральная машина
- Холодильник
- Телевизор
- Посудомоечная машина
- Мебель

### Пример 2: Продажа автомобиля

**Базовые атрибуты:**
- Марка: Список (динамический из базы)
- Модель: Связанный список (зависит от марки)
- Год выпуска: Число (мин: 1900, макс: текущий год)
- Тип кузова: Список [Седан, Хэтчбек, Универсал, Внедорожник и т.д.]
- Пробег: Число с единицей измерения "км"

**Технические характеристики:**
- Объем двигателя: Число с единицей измерения "л"
- Мощность: Число с единицей измерения "л.с."
- Тип топлива: Список [Бензин, Дизель, Электро, Гибрид, Газ]
- Привод: Список [Передний, Задний, Полный]
- Коробка передач: Список [Механическая, Автоматическая, Робот, Вариатор]

**Дополнительная информация:**
- Количество владельцев: Число
- ПТС: Список [Оригинал, Дубликат]
- Состояние: Список [Отличное, Хорошее, Среднее, Требует ремонта]
- Возможен торг: Булево значение
- Возможен обмен: Булево значение

### Пример 3: Услуги репетитора

**Базовые атрибуты:**
- Предмет: Список [Математика, Физика, Химия, Иностранные языки и т.д.]
- Уровень обучения: Список с множественным выбором [Начальная школа, Средняя школа, Старшая школа, Подготовка к ЕГЭ, Университет]
- Формат занятий: Список с множественным выбором [Индивидуально, Группа, Онлайн, У преподавателя, У ученика]
- Стоимость: Число с валютой + выбор [За занятие, За месяц, За курс]

**Дополнительная информация:**
- Образование: Текст
- Опыт работы: Число с единицей измерения "лет"
- Возрастные группы: Список с множественным выбором [Дошкольники, Школьники, Взрослые]
- Расписание: Специальный тип "Расписание" (дни недели и часы)
- Место проведения: Текст + карта (условно, если занятия не онлайн)

## Лучшие практики и советы

1. **Используйте стандартные атрибуты**, когда это возможно, для обеспечения единообразия данных и удобства поиска.

2. **Разделяйте атрибуты на обязательные и необязательные** осмысленно. Слишком много обязательных полей может отпугнуть пользователей.

3. **Группируйте связанные атрибуты** в логические блоки для улучшения пользовательского опыта.

4. **Используйте зависимые атрибуты** для создания динамических форм (например, выбор модели автомобиля в зависимости от марки).

5. **Регулярно анализируйте пользовательское поведение** для оптимизации набора атрибутов. Если какой-то атрибут редко заполняется, возможно, он не нужен.

6. **Проверяйте форму** после настройки атрибутов, чтобы убедиться в удобстве пользовательского интерфейса.

7. **Обеспечьте полный перевод** всех названий атрибутов и значений для поддерживаемых языков.

8. **Используйте подсказки и описания** для сложных атрибутов, чтобы помочь пользователям правильно заполнить форму.

9. **Обновляйте списки значений** по мере необходимости (например, добавляйте новые марки и модели автомобилей).

10. **Тестируйте каждую форму** с разными наборами данных перед публикацией.
